# PROJECT RULES FOR ANTIGRAVITY

## Absolute do-not-do list

- Do not start over.
- Do not migrate frameworks unless the verified current repo proves a different active architecture.
- Do not copy external repos into this project.
- Do not add Docker/FastAPI/MySQL/nginx unless already active and required.
- Do not add live trading execution.
- Do not add broker APIs.
- Do not scrape X/Twitter/Instagram/YouTube/login-gated pages.
- Do not add API keys or secrets.
- Do not invent live AI claims.
- Do not leave broken imports.
- Do not leave duplicate route systems.
- Do not fake passing validation.

## Required workflow

1. Read repo context files.
2. Detect stack and active app.
3. Run current validation/build.
4. Fix blockers.
5. Stabilize routes.
6. Consolidate duplicate data/layout.
7. Apply premium visual polish.
8. Build bookmark intelligence only after stability.
9. Run validation again.
10. Report exactly what changed.
