# Alert Story System

Signals should look and read like blueprints.

## Purpose

An alert should tell the story of the trade idea before the user ever opens Webull.

## Blueprint Sections

- Setup: what is happening
- Trigger: what must confirm
- Contract: exact option to inspect
- Risk: what invalidates the idea
- Target: what success looks like
- Webull: manual verification and execution location

## Visual Direction

Use dark blueprint panels, linework, grid texture, compact stats, and clear risk gates. The design should feel custom and strategic, not like a generic notification.
