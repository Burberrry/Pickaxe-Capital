# Pickaxe Capital Coding Guidelines

Read `docs/vision/MASTER_CONTEXT.md` first before architectural, UI, or engineering decisions.

## Principles

1. Think before coding.
2. Simplicity first.
3. Surgical changes only.
4. Goal-driven execution.

## Project Rules

- This is a synchronized AI operating system, not a widget collection.
- Optimize for interconnected intelligence, premium futuristic UX, living-system behavior, and market usefulness.
- Do not overengineer.
- Do not refactor unrelated files.
- Use mock data first when real agent data is not ready.
- Keep trading execution outside the website. Webull remains the manual execution venue.
